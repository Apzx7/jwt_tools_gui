# 安全政策

## 支持的版本

本项目仅支持最新版本的代码。对于之前版本的修复将根据情况决定。

## 报告漏洞

如果您发现安全漏洞，请不要在GitHub上公开创建issue。请发送邮件至 [your-email@example.com]，并提供以下信息：

- 漏洞类型
- 漏洞描述
- 复现步骤
- 潜在影响

我们会在收到报告后尽快回复，并在修复后发布安全更新。

## 安全最佳实践

本工具仅用于授权的安全测试。请在使用前确保：

1. 您有权限测试目标系统
2. 您的使用行为符合当地法律法规
3. 您不会将本工具用于恶意目的
4. 您了解JWT安全机制和相关风险
5. 您已获得适当的授权进行渗透测试

## 负责任的披露

我们承诺：

- 在收到漏洞报告后24小时内确认收到
- 在7个工作日内评估漏洞
- 在修复漏洞后及时发布安全更新
- 在漏洞修复前不公开披露漏洞详情
- 尊重研究者的隐私和贡献
